pytho——代码规范

一、编码

无特殊情况，使用 UTF-8编码
文件头部加入

二、缩进

使用4个空格进行缩进，或者 tab缩进

三、空格

  • 二元运算符两个各一个空格=,-,+=,==,>,in,is not, and
  • 函数参数列表中，后要加空格
  • 函数参数列表中，赋值语句=左右不加空格

四、空行

  • 模块奇函数和类之间空两行
  • 类成员函数之间空一行
  • 可以空行分割逻辑相关的代码
  • 可以用多个空行分割多组相关的函数

五、行宽

  • 每行代码最长不超过80，或者不超过120
  • 控制在80内的优势：
    • 这在查看 side-by-side 的 diff 时很有帮助
    • 方便在控制台下查看代码
    • 太长可能是设计有缺陷

六、换行

  • 括号内换行，第二行缩进4个空格，可以\换行
  • 每行一个语句
  • 循环语句一定要换行

七、引号

  • 自然语言使用双引号
  • 机器表示使用单引号
  • 正则表达式使用 r” “
  • 文档(docstring) 使用三引号””” “””

八、注释

  • #后空一个
  • 重要的注释，# ===== #======
  • 文档注释，””” “””

九、import 语句

  • import 语句分行写
  • import 语句应使用 absolute import
  • import 语言放在头部，模块说明 docstring 之后
  • 可以用空行分割相关模块

git-基本操作（二）

1. 工作区与暂存区

为什么 Git 比其他版本控制系统设计得优秀，因为 Git 跟踪并管理的是修改，而非文件。 git commit 只负责把暂存区的修改提交了。 每次修改，如果不 add 到暂存区，那就不会加入到 commit 中。 主要注意下面三个命令

2. 撤销修改

命令git checkout – readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

总之，就是让这个文件回到最近一次git commit或git add时的状态。

3. 删除文件

一是确实要从版本库中删除该文件，那就用命令git rm删掉

git checkout其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都可以“一键还原”。
