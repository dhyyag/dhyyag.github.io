monkey测试

一、what is monkey testing？

Monkey：是 Android SDK提供的一个命令行工具，可以运行在任何版本的模拟器里或实际设备里。monkey 向系统发送伪随机的用户事件流（比如：按键输入，触摸屏幕输入，手势输入），从而对应用程序进行压力测试。
Monkey testing：是Android自动化测试、压力测试的一种手段.

二、How to execute monkey testing？

2.1 软硬件环境准备

硬件：电脑，Android手机（已安装被测 app），数据线
软件：android SDK，Android 模拟器（安装被测 app），adb 命令可用

2.2 常规类命令

  • adb install -r -t 要测试的应用的 apk包的路径
    安装 apk 包到手机
  • adb logcat | grep START
    查询待测试的 app 的包名，查询 cmp，cmp 对应的值就是包名 比如：cmp=com.liulishuo.lingodarwin
  • adb shell monkey -p com.liulishuo.lingodarwin 1000
    -p指定测试对象包名
    1000mon
  • adb shell monkey -v -p com.liulishuo.lingodarwin 1000
    -v:打印日志信息
    -v -v: 打印比较详细的日志信息
    -v -v -vg:打印更详细的日志信息
    1000:monkey向手机上发送1000个随机事件

命令执行结束后，log中会显示Events injected: 1000，表示执行的随机事件数,monkey向手机上的com.liulishuo.lingodarwin应用程序发送1000个随机事件

  • adb shell monkey -c <main-category> 100
    -c指定测试一个或多个类别
    <main-category>指定的类别，比如：Inter.CATEGORY_LAUNCHER

2.3 事件类命令

  • adb shell monkey -f 脚本的路径 该脚本循环次数
    -f表示 monkey 运行指定路径下的脚本
  • adb shell monkey -v -p com.liulishuo.lingodarwin -s 100 1000
    -s指定 seed 值(可重复执行刚才的随机操作）
    100指定随机数生成器的 seed 值，
    10001000g个随机事件
  • adb shell monkey -v -p com.liulishuo.lingodarwin --throttle 500 100
    --throttle表示事件事件的的固定延迟事件，单位 ms，设置为500比较接近人的操作
  • adb shell monkey -v -p com.liulishuo.lingodarwin --pct-touch 50 100
    --pct-touch:接触摸事件
    50:触摸事件占比50%

2.4 调试类命令

  • adb shell monkey --ignore-crashes 1000
    --ignore-crashes:当应用程序崩溃或发生失控异常时，monkey将继续运行直到计数完成。如果不设置此项，monkey遇到上述崩溃或异常将停止运行。
  • adb shell monkey --ignore-timeouts 1000
    --ignore-timeouts:当应用程序发生任何超时错误（如ANR，即Application Not Responding)时，monkey将继续运行直到计数完成。如果不设置此项，monkey遇到此类超时将停止运行
  • adb shell monkey --kill-process-after-error 1000
    --kill-process-after-error:当monkey因为应用程序发生错误而停止时，将会通知系统停止发生错误的进程。如果不设置此项，monkey停止时发生错误的应用程度将继续处于运行状态。

  • adb shell monkey --monitor-native-crashes 1000
    --monitor-native-crashes:monkey运行时native code的崩溃事件将被监视并报告。如果不设置此选项，将不会监视此类事件。

    三、Analyze monkey test results

    测试结束后，需要将手机连接PC，拷贝以下文件,我们需要对这3个文件进行分析整理，以便提交开发人员处理。

  • info.txt：此文件在手机上的SDCARD中，主要记录了MONKEY测试时发送的各种事件，如触摸事件的位置等等。

  • error.txt：此文件在手机上的SDCARD中，主要记录了MONKEY测试时产生的一些ANR、强制关闭等异常。
  • LOG文件：此文件在手机上的LOG文件夹中中，主要主要记录程序对MONKEY测试时的响应情况。
